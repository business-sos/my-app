# Business Health Dashboard

A multi-tenant coaching dashboard organized around **The Five Questions of Business Health**:

1. How well are we growing our customer base? *(Customer Acquisition)*
2. How well are we looking after our customers? *(Customer Care)*
3. How strong is the team? *(Team Strength)*
4. How reliable are our systems? *(Systems Reliability)*
5. How strong is our financial position? *(Financial Position)*

Coaches invite clients; clients log in and connect their data (Google Sheet, manual entry, or P&L upload). The dashboard surfaces cross-area Insights generated by Claude from the actual numbers — each one with a dollar impact, evidence from the data, and a concrete action for the week.

## Stack

- **Frontend**: React 19 + Vite + Recharts + react-router
- **Auth + DB**: Supabase (Postgres with Row-Level Security for tenant isolation)
- **Serverless**: Vercel functions in `api/`
- **AI**: Anthropic Claude (`claude-sonnet-4-6`) — never called from the browser
- **Integrations**: Google Sheets (published-CSV or link-shared URLs), manual P&L upload (XLS/CSV)

## Local development

```bash
cd dashboard
cp .env.example .env.local      # fill in VITE_SUPABASE_*, ANTHROPIC_API_KEY, CRON_SECRET
npm install
npm run dev
```

Visit http://localhost:5174.

The dev server has a tiny built-in plugin (`dev-api-plugin.js`) that serves `/api/*` handlers as if they were Vercel functions — so the full stack runs from one terminal.

### One-time Supabase setup

1. Create a project at https://supabase.com.
2. Apply migrations in order (paste each into the SQL Editor or use `supabase db push`):
   - `supabase/migrations/0001_init.sql` — core schema + RLS
   - `supabase/seed.sql` — 5 areas + ~21 starter indicators
   - `supabase/migrations/0002_financial_analysis.sql` — P&L upload tables + Storage bucket
   - `supabase/migrations/0003_csv_imports.sql` — Google Sheet imports + mappings
   - `supabase/migrations/0004_csv_layout.sql` — long/wide layout support
   - `supabase/migrations/0005_csv_period_overrides.sql` — year-less header overrides
   - `supabase/migrations/0006_insights.sql` — cross-area Insights cache
   - `supabase/migrations/0007_csv_sync_runs.sql` — sync history per import
3. In Authentication → URL Configuration, set:
   - **Site URL**: your production URL (or `http://localhost:5174` for local dev).
   - **Redirect URLs**: add both `http://localhost:5174` and your production URL.
4. (Optional) Enable Google as an Auth provider for one-click sign-in.

### Becoming a coach

The signup trigger creates new users as `role = 'client'` by default. The `/signup` page passes `role = 'coach'` metadata so coaches created via the UI become coaches automatically. To promote an existing user:

```sql
-- Replace the email + practice name.
do $$
declare org_id uuid;
begin
  insert into organizations (name) values ('My Practice') returning id into org_id;
  update profiles
     set role = 'coach', organization_id = org_id, display_name = 'Coach'
   where id = (select id from auth.users where email = 'you@example.com');
end $$;
```

After that, the coach can use **+ Add client** in the UI for everything else.

## Deploying to Vercel

The dashboard ships as its own Vercel project — independent from the repo's existing apps.

### 1. Import the repo

- New Project → import the GitHub repo.
- **Root Directory**: `dashboard` (this is the critical setting).
- Framework preset: **Vite** (auto-detected).
- Build command: `npm run build`.
- Output directory: `dist`.

### 2. Environment variables

In Project Settings → Environment Variables, add the following. **Mark anything that is NOT prefixed with `VITE_` as server-only** (uncheck "Available in browser" if Vercel exposes that toggle).

| Name | Where | Notes |
|---|---|---|
| `VITE_SUPABASE_URL` | Production + Preview + Dev | `https://xxx.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | Production + Preview + Dev | publishable key |
| `SUPABASE_SERVICE_ROLE_KEY` | Production + Preview only | **server-only** — admin bypass |
| `ANTHROPIC_API_KEY` | Production + Preview | **server-only** — burned by `/api/llm-*` |
| `CRON_SECRET` | Production only | `openssl rand -hex 32` — Vercel sends as Bearer on cron calls |

Do **NOT** set `VITE_DEV_AUTOLOGIN_*` in production. The app has a hard guard that ignores them in PROD builds, but they shouldn't be there in the first place.

### 3. Deploy

Click Deploy. After it succeeds:

1. Note the production URL (e.g. `https://your-app.vercel.app`).
2. Go back to Supabase → Authentication → URL Configuration → add your production URL to both Site URL and Redirect URLs.
3. In the Vercel dashboard → Settings → Cron Jobs, confirm the entry `0 18 * * *  /api/csv-imports-cron` is present (auto-picked up from `vercel.json`).

### 4. First production check

- Visit `https://your-app.vercel.app` — should land on the Sign in page (auto-login is off in PROD even if the env vars were set).
- Sign up at `/signup` to create the first coach + practice.
- Add a client → invite by email → click the magic link in another browser → land in the client view.
- Connect a Google Sheet → run a sync → verify measurements land and Insights generate.

## Verification checklist (after deploy)

| Check | How |
|---|---|
| Schema + RLS | As coach, in the browser console: `await window.supabase.from('clients').select('*')` returns only your org's clients. |
| Auth | Magic-link sign-in works in incognito; redirected back to the app. |
| Coach signup | `/signup` creates a profile with role=coach and an organizations row. |
| Add client | `/` → + Add client → row appears immediately. With email, invite is sent. |
| Manual entry | `/entry` → pick indicators → enter values for 3 periods → reload → sparklines appear. |
| Insights | `/client/<id>` → InsightsPanel renders with 5–8 cards within ~10s on first generation; cached for subsequent loads. |
| Sheet import | `/integrations` → paste any Google Sheet URL → Detect → confirm → Save & sync → measurements land. |
| Sheet history | Integrations card → History → shows your manual + cron runs. |
| Cron | Vercel → Logs filter on `/api/csv-imports-cron` → daily 18:00 UTC entries with `{ succeeded, failed }`. |
| Financial analysis | `/financial` → upload a P&L → confirm extracted values → run analysis → ratios + narrative render. |

## Cost guardrails

- **Anthropic**: every Insights generation, narrative, extraction, or mapping call burns Claude tokens. At a 100-client × monthly cadence the per-month cost is ~$5–15. Set a budget alert in the Anthropic console.
- **Supabase**: free tier sleeps after 7 days of inactivity (we hit this during dev). Upgrade to Pro ($25/mo) when you have real users.
- **Vercel**: hobby plan is fine for the first 5–10 clients. Pro for cron + team features.

## Skipped, on-purpose

- **Google Sheets OAuth + Drive API**: not needed for the published-CSV path. Wired up but unused; leave the OAuth env vars unset in production.
- **`api/sheets-sync.js`**: legacy from the OAuth-Sheets era. The active pipeline is `csv-import-sync.js` + `csv-imports-cron.js`.
- **Dev auto-login** (`VITE_DEV_AUTOLOGIN_*`): local dev only. Hard-disabled in PROD builds.
